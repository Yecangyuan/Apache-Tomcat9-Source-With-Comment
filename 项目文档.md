## 一、项目概览

### 1.1 基本信息

| 属性 | 值 |
|------|-----|
| **项目名称** | Apache Tomcat |
| **版本** | 9.0.90 |
| **定位** | 开源 Java Servlet 容器，实现了 Java Servlet、JSP、EL 和 WebSocket 规范 |
| **开发语言** | Java |
| **构建工具** | Apache Ant |
| **许可证** | Apache License 2.0 |
| **源码文件数** | 约 1,802 个 Java 文件 |

### 1.2 项目简介

Apache Tomcat 是一个开源的轻量级 Web 应用服务器，由 Apache 软件基金会开发和维护。它是 Java EE 规范中 Servlet、JSP (JavaServer Pages)、EL (Expression Language) 和 WebSocket 技术的官方参考实现。

Tomcat 广泛应用于生产环境，为众多大规模、关键任务的 Web 应用程序提供支持。

### 2.2 核心组件说明

Tomcat 的核心架构遵循**分层设计**，主要分为以下几层：

#### 2.2.1 Server (服务器)
- **定义**：整个 Tomcat 实例的顶层容器
- **职责**：
  - 管理一个或多个 Service
  - 提供全局命名资源 (JNDI)
  - 监听关闭端口 (默认 8005)，接收 SHUTDOWN 命令
- **实现类**：`org.apache.catalina.core.StandardServer`

#### 2.2.2 Service (服务)
- **定义**：将一个或多个 Connector 与一个 Engine 关联
- **职责**：
  - 组合 Connector 和 Container
  - 统一管理生命周期
- **关系**：一个 Server 可包含多个 Service，一个 Service 包含一个 Engine 和多个 Connector

#### 2.2.3 Connector (连接器)
- **定义**：处理客户端连接的组件
- **职责**：
  - 监听指定端口接收请求
  - 将请求转换为标准 Request/Response 对象
  - 支持多种协议：HTTP/1.1、HTTP/2、AJP/1.3
- **实现类**：`org.apache.catalina.connector.Connector`
- **协议处理器**：`org.apache.coyote.ProtocolHandler`

#### 2.2.4 Container (容器)

容器采用**责任链模式**组织，形成层级结构：

| 容器 | 说明 | 类比 |
|------|------|------|
| **Engine** | 整个 Servlet 引擎，处理所有请求 | 引擎 |
| **Host** | 虚拟主机，代表一个域名 | 域名/主机 |
| **Context** | Web 应用，对应一个 WAR 包 | 应用 |
| **Wrapper** | 单个 Servlet 的包装器 | Servlet |

### 2.3 请求处理流程

```
1. 客户端请求
        │
        ▼
2. Connector 接收请求 (Coyote)
        │
        ▼
3. 生成 Request/Response 对象
        │
        ▼
4. 传递给 Engine
        │
        ▼
5. 根据域名匹配 Host
        │
        ▼
6. 根据路径匹配 Context (Web应用)
        │
        ▼
7. 根据 URL 匹配 Wrapper (Servlet)
        │
        ▼
8. 调用 FilterChain → Servlet.service()
        │
        ▼
9. 返回响应给客户端
```

## 五、关键技术点

### 5.1 Pipeline-Valve 机制

Tomcat 使用**责任链模式**处理请求，通过 Pipeline 和 Valve 实现：

```java
// Valve 接口 - 请求处理链中的节点
public interface Valve {
    // 获取下一个 Valve
    Valve getNext();
    
    // 设置下一个 Valve
    void setNext(Valve valve);
    
    // 处理请求
    void invoke(Request request, Response response) 
        throws IOException, ServletException;
}

// Pipeline 接口 - 管理 Valve 链
public interface Pipeline {
    // 获取基础 Valve (最后一个)
    Valve getBasic();
    
    // 设置基础 Valve
    void setBasic(Valve valve);
    
    // 添加 Valve
    void addValve(Valve valve);
    
    // 移除 Valve
    void removeValve(Valve valve);
}
```

**标准 Valve 链**：

```
AccessLogValve ──► ErrorReportValve ──► StandardHostValve 
                                                      │
                                                      ▼
                                      StandardContextValve
                                                      │
                                                      ▼
                                       StandardWrapperValve
                                                      │
                                                      ▼
                                              Servlet
```

### 5.2 类加载机制

Tomcat 实现了自定义的类加载器层次：

```
Bootstrap ClassLoader (JVM)
         │
         ▼
System ClassLoader (CLASSPATH)
         │
         ▼
Common ClassLoader (CATALINA_HOME/lib)
         │
    ┌────┴────┐
    ▼         ▼
Shared   Catalina
         │
         ▼
    WebApp ClassLoader (WEB-INF/classes, WEB-INF/lib)
```

**特点**：
- 每个 Web 应用有独立的类加载器
- 实现类隔离，避免冲突
- 支持热部署 (热加载)

### 5.3 Session 管理

Tomcat 提供多种 Session 管理器：

| 管理器 | 说明 |
|--------|------|
| `StandardManager` | 标准实现，Session 保存在内存 |
| `PersistentManager` | 支持持久化到存储 |
| `DeltaManager` | 集群中广播 Session 变更 |
| `BackupManager` | 集群中备份 Session |

### 5.4 集群与高可用

```
┌─────────────┐      组播/复制       ┌─────────────┐
│  Tomcat A   │◄──────────────────►│  Tomcat B   │
│  (Node 1)   │                    │  (Node 2)   │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       └──────────┬───────────────────────┘
                  │
           Apache Tribes
         (组通信框架)
```

**Tribes 组件**：
- 提供可靠的组通信
- 支持 DeltaManager (增量复制)
- 支持 BackupManager (备份复制)

## 七、总结

### 7.1 设计亮点

1. **模块化设计**：各模块职责清晰，耦合度低
2. **可扩展性**：通过 Valve、Realm、Manager 等接口支持扩展
3. **配置灵活**：支持 XML 配置和嵌入式 API
4. **性能优化**：NIO、线程池、缓冲区复用等
5. **标准兼容**：完整实现 Servlet 4.0、JSP 2.3 等规范

### 7.2 适用场景

- Java Web 应用部署
- 微服务架构中的 Servlet 容器
- 嵌入式 Web 服务器
- 开发测试环境

### 7.3 学习价值

- 深入理解 Servlet 容器工作原理
- 学习网络编程和协议处理
- 掌握多线程和并发处理
- 了解软件架构设计模式
